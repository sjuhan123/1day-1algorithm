### 분석

in-place, Two Pointer 문제

**in-place**

1. 추가적인 자료구조를 사용하지 않는다. 
2. 주어진 입력 공간 안에서 해결한다.
3. 주어진 배열을 직접 수정한다.
4. 새로운 배열을 만들지 않는다.

**Two Pointer**

하나의 배열을 두 개의 인덱스로 나눠서 서로 다른 역할을 맡겨 처리하는 기법\n
=> 읽는 포인터와 쓰는 포인터를 분리

대표 유형 3개

1. READ / WRITE 패턴

```js
var write = 0;
for(let read = 0; read < legnth; read++) {
    if(조건) {
        nums[write] = nums[read]
    }
}
```

2. 양 쪽에서 좁히는 패턴

```js
let i = 0; j = n - 1;

while(i < j) {
    if(조건) i++
    else j--
}
```

3. 슬라이딩 윈도우(Two Pointer 확장)

```js
let k = 0;
for(let i = 0; i < n; i++) {
    while(조건 위반) k--
}
```
